# Call Recorder App Changes

## v1.0.6
- Storage path always resolves to /storage/emulated/0/<user-path>
- Settings displays and saves the full absolute path
- Browse button auto-prepends /storage/emulated/0/DCIM/ to selected path

## v1.0.5
- In-app recordings list screen showing all saved files with name, date, size, format
- Tapping a recording opens it or offers to share via audio player
- Default storage: /storage/emulated/0/DCIM (configurable in settings)
- Browse button in settings opens SAF folder picker to choose storage location
- Scans DCIM/, DCIM/Recordings/, Music/CallRecorder/ for all recordings

## v1.0.4
- Added auto-record toggle: automatically start recording when a call is answered/off-hook
- Calls stop recording automatically when the call ends (state goes idle)
- Respects included/excluded contact filter settings
- CallReceiver monitors phone state via PHONE_STATE intent
- Toggle in Settings: "Auto-record when call answered"

## v1.0.3
- Fixed version checker to point to correct GitHub repo (jnetai-clawbot/call-recorder)
- Version info matches GitHub release tag (v1.0.3)
- Same keystore as v1.0.2 for seamless updates

## v1.0.2
- Dual source recording: microphone + speaker capture with per-source volume controls
- Output format selection: WAV (default), FLAC, MP3, AAC, OGG/Vorbis with codec fallback
- Selectable microphone audio sources: Call (Mic+Speaker), Microphone, Camcorder, Voice Recognition, Unprocessed
- Contact filtering: included/excluded contacts for recording (default: all contacts recorded)
- Auto-start on boot with settings toggle (enabled by default)
- BootReceiver for RECEIVE_BOOT_COMPLETED intent
- PulseView animated recording indicator
- Updated AudioEngine with professional mixing, volume normalization, and multi-format encoding
- Persistent error handling with error codes throughout all components
- Default storage location: DCIM/Recordings/call-recording-datetime.ext
- RecorderService loads configuration from settings
- Comprehensive settings UI with sliders, spinners, toggles, and contact picker
- About activity with GitHub release checker and version display
- Dark theme with professional layout and bottom spacing

## Initial Version
- Full Android app structure with Gradle/Kotlin
- AudioEngine with WAV recording support (PCM 16-bit, 44.1kHz mono)
- RecorderService as foreground service with notification controls
- MainActivity with recording timer and status display
- SettingsActivity with format selection (WAV/FLAC/MP3)
- AboutActivity with GitHub release version checker
- Dark theme throughout (Material 3 Dark)
- FileProvider for sharing recordings
- RIFF WAV header writing with proper formatting
- Error handling with error codes: CR-AE, CR-RS, CR-MA, CR-SE, CR-AB
- GitHub Actions CI/CD workflow for building release APK
- Same keystore for all builds (ciphervault-release-keystore.jks)
- APK output named Call-Recorder.apk
